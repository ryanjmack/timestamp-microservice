Unix Timestamp Microservice

Documentation

  1. The API endpoint is /api/timestamp/:date_string?
  2. A date string is valid if it is one of the following:
    • An ISO-8601 date respresentation in the format YYYY-MM-DD (Ex: 2018-12-25)
    • The number of seconds after the Unix Epoch (Ex: 1545696000)
  3. If the datestring is empty, the currrent timestamp will be used.
  4. If the date is valid it will return JSON in the format: {"unix": "Unix Timestamp", "utc": "UTC String"}
  5. If the date is invalid it will return JSON in the format: {"error": "Invalid Date"}

Examples of usage

api/timestamp/2018-12-25

api/timestamp/1545696000

Example Output

{"unix": "1545696000", "utc": "Tue, 25 Dec 2018 00:00:00 GMT"}
